# Best Practices for Digital Signage

## Design Principles

### 1. The 10-Foot Rule

Content must be readable from at least 10 feet away:

- **Minimum font sizes**: 48px for body text, 72px+ for headings
- **High contrast**: Light text on dark backgrounds or vice versa
- **Bold weights**: Medium (500) minimum, Bold (700) preferred
- **Avoid**: Small text, low contrast, thin fonts

### 2. Fixed Aspect Ratios

Always design for specific resolutions:

- **1080p landscape**: 1920x1080 (16:9)
- **4K landscape**: 3840x2160 (16:9)
- **1080p portrait**: 1080x1920 (9:16)

Use `ScreenFrame` to enforce these constraints during development.

### 3. Deterministic Layouts

Signage layouts must be predictable:

- **No dynamic reflows**: Content shouldn't shift based on data length
- **Fixed heights**: Use absolute heights, not `auto`
- **Text clamping**: Use `line-clamp` to prevent overflow
- **Safe areas**: Keep critical content within safe margins

### 4. Performance Considerations

Signage runs 24/7 on embedded hardware:

- **Optimize images**: Compress and use appropriate formats (WebP)
- **Limit animations**: Smooth 60fps animations only
- **Avoid memory leaks**: Clean up intervals, timeouts, and event listeners
- **Test on target hardware**: BrightSign devices have different performance characteristics

## Component Usage

### ScreenFrame

Always wrap signage content in `ScreenFrame`:

```tsx
<ScreenFrame resolution="1080p" scale={0.5}>
  {/* Your signage content */}
</ScreenFrame>
```

- Use `scale={0.5}` during development for easier preview
- Set `scale={1}` or omit for production
- Enable `showSafeArea` during QA to verify margins

### Layout Patterns

#### SplitScreen

Use for two-zone layouts:

```tsx
<SplitScreen direction="row" ratio="60-40">
  <MainContent />
  <Sidebar />
</SplitScreen>
```

**Best for**:

- Dashboard + sidebar
- Hero + callout
- Main content + branding

#### FullscreenHero

Use for single-message displays:

```tsx
<FullscreenHero
  title="Welcome"
  subtitle="Modern workspace"
  variant="dark"
  backgroundImageUrl="/hero-bg.jpg"
/>
```

**Best for**:

- Lobby welcome screens
- Event announcements
- Branding displays

#### InfoCardGrid

Use for metrics and data displays:

```tsx
<InfoCardGrid items={metricCards} columns={3} density="comfortable" />
```

**Best for**:

- KPI dashboards
- Status boards
- Real-time metrics

## Typography

### Size Scale

Use these sizes for signage content:

- **Small**: 48px
- **Base**: 56px
- **Large**: 72px
- **XLarge**: 96px
- **Hero**: 144px+

### Font Weights

- **Body text**: 500 (medium) minimum
- **Headings**: 600 (semibold) or 700 (bold)
- **Emphasis**: 700 (bold) or 800 (extrabold)

### Line Height

- **Headings**: 1.1 to 1.2
- **Body text**: 1.3 to 1.5
- **Never**: Less than 1.1 (causes cramped text)

## Color and Contrast

### WCAG AAA Compliance

Signage should meet WCAG AAA standards (7:1 contrast ratio):

- **Dark backgrounds**: Use `slate-900` or darker
- **Light text**: Use `white` or `slate-50`
- **Avoid**: Mid-tone backgrounds with mid-tone text

### Color Usage

- **Primary accent**: Use sparingly for emphasis
- **Status colors**: Green (positive), red (negative/alert), yellow (warning)
- **Backgrounds**: Dark (preferred) or very light, avoid mid-tones

## Common Pitfalls

### 1. Dynamic Content Overflow

❌ **Don't**: Allow dynamic text to break layout

```tsx
<div>
  <h1>{dynamicTitle}</h1> {/* Could be any length! */}
</div>
```

✅ **Do**: Use text clamping

```tsx
<div>
  <h1 className="line-clamp-2">{dynamicTitle}</h1>
</div>
```

### 2. Responsive Breakpoints

❌ **Don't**: Use responsive breakpoints for signage

```tsx
<div className="text-base md:text-lg lg:text-xl">
  {/* This is for websites, not signage */}
</div>
```

✅ **Do**: Use fixed sizes for known resolutions

```tsx
<div className="text-6xl">
  {' '}
  {/* Fixed size for signage */}
  Content
</div>
```

### 3. Auto Heights

❌ **Don't**: Use auto heights that can cause reflow

```tsx
<div className="h-auto">
  {dynamicContent} {/* Height changes = layout shift */}
</div>
```

✅ **Do**: Use fixed or calculated heights

```tsx
<div className="h-[400px] overflow-hidden">{dynamicContent}</div>
```
