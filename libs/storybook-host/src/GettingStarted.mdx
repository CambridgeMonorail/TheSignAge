# Getting Started

## Installation

The Sign Age component libraries are organized in an Nx monorepo. To use them in your project:

### 1. Clone the Repository

```bash
git clone https://github.com/CambridgeMonorail/TheSignAge.git
cd TheSignAge
```

### 2. Install Dependencies

```bash
pnpm install
```

### 3. Run Storybook Locally

```bash
pnpm storybook
```

This will start Storybook at `http://localhost:4400` where you can explore all components interactively.

### 4. Build the Client App

```bash
pnpm serve:client
```

This starts the demo website at `http://localhost:4173`.

## Using Signage Components

### Import from @tsa/shadcnui-signage

```typescript
import {
  ScreenFrame,
  SignageContainer,
  FullscreenHero,
  InfoCardGrid,
  SplitScreen,
} from '@tsa/shadcnui-signage';
```

### Basic Example: Fullscreen Hero

```tsx
import { ScreenFrame, FullscreenHero } from '@tsa/shadcnui-signage';

export function MySignage() {
  return (
    <ScreenFrame resolution="1080p" scale={0.5}>
      <FullscreenHero
        title="Welcome to Our Lobby"
        subtitle="Modern workspace for modern teams"
        variant="dark"
      />
    </ScreenFrame>
  );
}
```

### Example: Split Screen Layout

```tsx
import { ScreenFrame, SplitScreen } from '@tsa/shadcnui-signage';

export function DashboardSignage() {
  return (
    <ScreenFrame resolution="1080p">
      <SplitScreen direction="row" ratio="60-40">
        {/* Left zone: Main content */}
        <div className="p-8">
          <h1 className="text-6xl font-bold">Dashboard</h1>
        </div>

        {/* Right zone: Sidebar */}
        <div className="p-8 bg-slate-900">
          <h2 className="text-4xl">Stats</h2>
        </div>
      </SplitScreen>
    </ScreenFrame>
  );
}
```

### Example: Info Card Grid

```tsx
import { ScreenFrame, InfoCardGrid } from '@tsa/shadcnui-signage';

const cards = [
  {
    title: 'Revenue',
    value: '$1.2M',
    subtitle: '+12% from last month',
    status: 'positive' as const,
  },
  {
    title: 'Active Users',
    value: '12,345',
    subtitle: '+234 this week',
    status: 'positive' as const,
  },
  // ... more cards
];

export function MetricsSignage() {
  return (
    <ScreenFrame resolution="1080p">
      <InfoCardGrid items={cards} columns={3} density="comfortable" />
    </ScreenFrame>
  );
}
```

## Understanding ScreenFrame

`ScreenFrame` is a foundational primitive that ensures your signage layouts are deterministic and preview correctly:

- **resolution**: Target screen resolution (`'1080p'`, `'4k'`, `'portrait-1080p'`)
- **scale**: Preview scale factor (0.1 to 1.0) for development
- **showSafeArea**: Toggle safe area overlay for QA

Always wrap your signage content in a `ScreenFrame` to ensure correct aspect ratios and responsive behavior.

## Introduction

Welcome to the TheSignAge (TSA) monorepo. This document provides an overview of the various libraries available within the monorepo, each designed to help you build a consistent, maintainable, and cohesive application.

## Overview of Libraries

#### 1. **`@tsa/shadcnui`**

- **Description**: A library providing base UI components styled with @tsa/shadcnui, ensuring a consistent look and feel across the application. It includes components like switches, tables, and more.
- **Usage**: Use these components to maintain a cohesive design system throughout the application.
- **Example**:

  ```tsx
  import { Switch, Table, TableRow, TableCell } from '@tsa/shadcnui';

  const Settings = () => (
    <div>
      <Switch checked={true} onCheckedChange={() => {}} />
      <Table>
        <TableRow>
          <TableCell>Item 1</TableCell>
          <TableCell>Item 2</TableCell>
        </TableRow>
      </Table>
    </div>
  );
  ```

For more information on @tsa/shadcnui, visit their [website](https://ui.shadcn.com/).

#### 2. **`@tsa/shadcnui-blocks`**

- **Description**: A collection of composite UI components built from the base Shadcn UI components. The library includes components such as buttons, cards, dialogs, and more.
- **Usage**: Import and use these components to build consistent and cohesive user interfaces.
- **Example**:

  ```tsx
  import { Button, Card, Dialog } from '@tsa/shadcnui-blocks';

  const App = () => (
    <div>
      <Button variant="primary">Click Me</Button>
      <Card title="Card Title">Card Content</Card>
      <Dialog isOpen={true} onClose={() => {}}>
        Dialog Content
      </Dialog>
    </div>
  );
  ```

#### 3. **`@tsa/common-tailwind`**

- **Description**: A library containing common Tailwind CSS configurations and utility classes used across the monorepo. It includes preflight styles and theme configurations.
- **Usage**: Import the CSS files to apply consistent styling and utility classes throughout the application.
- **Example**:
  ```css
  @import '@tsa/common-tailwind/src/preflight.css';
  @import '@tsa/common-tailwind/src/shadcn-theme.css';
  ```

For more information on Tailwind CSS, visit their [website](https://tailwindcss.com/).

#### 4. **`@tsa/landing`**

- **Description**: A library containing components and sections specifically designed for the landing pages of the application. It includes sections like Hero, About, CTA, and more.
- **Usage**: Use these components to build engaging and informative landing pages.
- **Example**:

  ```tsx
  import { HeroSection, AboutSection, CTASection } from '@tsa/landing';

  const LandingPage = () => (
    <div>
      <HeroSection title="Welcome" subtitle="Discover our platform" />
      <AboutSection />
      <CTASection />
    </div>
  );
  ```

#### 5. **`@tsa/shell`**

- **Description**: A library providing layout components and utilities for the overall structure of the application. It includes components like `Layout`, `AppSidebar`, `Breadcrumb`, and more.
- **Usage**: Use these components to create a consistent and responsive layout for the application.
- **Example**:

  ```tsx
  import { Layout, AppSidebar, Breadcrumb } from '@tsa/shell';

  const App = () => (
    <Layout>
      <AppSidebar />
      <Breadcrumb />
      <main>{/* Main content */}</main>
    </Layout>
  );
  ```

### Summary

The monorepo is organized into several libraries, each serving a specific purpose to ensure a modular, maintainable, and cohesive codebase. These libraries include:

- **`@tsa/shadcnui`**: Base @tsa/shadcnui components.
- **`@tsa/shadcnui-blocks`**: Composite UI components built from @tsa/shadcnui components.
- **`@tsa/common-tailwind`**: Common Tailwind CSS configurations.
- **`@tsa/landing`**: Components for landing pages.
- **`@tsa/shell`**: Layout components and utilities.

By leveraging these libraries, you can build a consistent and maintainable application with a clear separation of concerns.

For more information on Nx, visit their [website](https://nx.dev/).
